A tentative object-oriented Limit

Mathematics

Pedro O S
January/2021

This is a tentative explanation for a Limit in a more computational way.

We have talked about how the $\delta, \varepsilon$ definition of continuity [1] expresses an infinitesimal concept in a disguised, unreadable, and anti-didactic way. This permeates the traditional pedagogy of analysis by consequence, as evidenced by cases where proofs are enunciated in error or in imprecise ways because the relationship between the parts of the statement are not understood by the presenter. But this is not the point here.

The infinitesimal in the $\delta, \varepsilon$ definition of limit can be found in:

$\forall \varepsilon \gt 0$ together with $x \lt \varepsilon$: we're looking for $x$ to be greater than $0$ and smaller than any real number greater than $0$; in other words, an infinitely small quantity.

Here is what a limit is:

$$ {\color{fuchsia}x' \gt x \Rightarrow} \, {\color{orange}0 \lt} \, {\color{red}\mid f(x) - L \mid} \, {\color{orange}\lt \varepsilon, \, \forall \varepsilon \in \mathbb{R}} {\color{fuchsia}\text{, and } x \text{ exists}}\label{1}\tag{1} $$
  • ${\color{fuchsia}x' \gt x\text{, }x\text{ exists}}$: for all elements in the domain of the function larger than some element $x$, which exists
  • ${\color{orange}0 \lt a \lt \varepsilon, \forall \varepsilon \in \mathbb{R}}$: $a$ is an infinitesimal; in the case above, the distance between the function value at the taken point and the proposed limit (explained below) is infinitesimal
  • ${\color{red}\mid f(x) - L \mid}$: the distance between the function value at element $x$ and the proposed limit $L$

So:

$L$ is the limit of a function if there exists some element of its domain such that for all elements larger than it (following it), the difference between the function value there and the proposed limit is a number smaller than any real number, but larger than zero. (That is, an infinitesimal.)

Or, in other words:

$L$ is the limit of a function if "past a certain point" in the function, the function is always infinitely close to $L$.

It is tempting, when trying to picture the above assertion, to think of it as describing the general evolution of the function along its domain; or from some arbitrary point of it ($x'$) onwards. This is incorrect; the above behavior is only true infinitely close to the "end" of the function's domain.

The "end", in case, is where the limit "happens": where the value of the function stabilizes into the limit.

This "place", interestingly, is at infinity: this means it can't be captured in a graph. Only the behavior "leading up to there" appears in the graph: the corresponding asymptote. But, again, the asymptote is not part of the above behavior for the existence of a limit. The asymptote is, with luck, an indicator of the presence of a limit; but in an arbitrarily defined function, an asymptote-looking curve could be followed by a non-limit. And, likewise, an irregular function "up to a point" could "end abruptly" in a limit, in a way that the limit's asymptote is small relative to the irregularity such that it can't be easily perceived in a graph.

The function stabilization, as we'll see, denotes, computationally, the termination of the enumeration of the function, albeit after an infinite amount of time; nevertheless, it is termination, because the function value is a constant after some point, and constants require no computation.

This also means enumeration of a function with a limit can't be non-terminating anywhere, and so this establishes the synonyms $\text{convergent}=\text{limited}=\text{eventually terminating}$ and $\text{divergent}=\text{non-terminating}$. What about $\text{terminating}$ (in a finite amount of time)? As we'll see, this corresponds to finite functions, that is, functions defined over finite domains that produce no non-finite elements out of those domain elements. Numerically, all functions over finite $\mathbb{N}$ subsets would classify as such.

Object aspects

Let's go over the descriptive elements used in the definition $\eqref{1}$.

  1. The relation of being smaller or larger than another element; that is, the order ($\lt$) relation
  2. The property of being an infinitesimal, that is, infinitely small
  3. The value of the distance between two elements

Of these, a few comments can be made.

Order

This is one of the problems in the traditional conception. The order relation is used to hold under the same class irrational and rational numbers (but not accepting infinitesimal ones, which is weird). To have this relation hold is why an infinitesimal class of numbers is not accepted, because those would not be directly comparable with the real numbers (further explanation of the problem of comparability will be given in a future article). We'll mention here the difference of behavior of $\lt$ between rationals and irrationals.

The relation of $\lt$ in the rationals is verifiable. Two segments with rational lengths can be measured to verify which of the two is smaller: measurement terminates. Contrast this to comparing a rational with an irrational, or two irrational-length segments: measurement of the irrational segment doesn't terminate, so $\lt$ can't be computationally verified when one of the compared numbers is not rational.

We can stipulate that if at some digit the irrationals or irrational and rational diverge, than the one number with the greatest digit is the greatest number. But that's an axiom in the system, introduced to decide in a finite time. With rationals, the order can be decided in finite time by simply measuring the numbers, and the axiom is not needed. This is a logical difference between the systems.

For our system, this means in the presence of an inconsistency, we're free to rethink: we can see order as generally any transitive difference, for instance (where transitivity is our familiar property of $a \, R \, b \land b \, R \, c \Rightarrow a \, R \, c$). Whereas 'order' is generally taken as to be a property exclusive to numbers, 'transitive difference' is appliable to a more general class of objects, while acceptably reducing to 'order' for the particular case of numbers (smaller or larger numbers are transitively different numbers).

As such, we'll do away with order and implement, generally, transitive difference.

Infinitesimal

In our conception, Infinitesimal is a class of numbers.

This conception is close to the Leibnizian conception of infinitesimals, or to non-standard analysis or other systems ([1], Addendum (Dec. 20)). In these systems, infinitesimals are smaller than any Real number; but are not only one between all them; there are infinite infinitesimals of different "sizes", and they can be compared, and put in ratios (just not together with Real numbers). Thus, Infinitesimals form a type which is not a subtype of Number.

In a way, such multiple infinitesimals can be analogized to the multiplicity of infinities, which is very real in Mathematics. They can be taken as reciprocals of (countable) infinities; the exact behavior of such a system will be elaborated in a system called a Fractional Space. We can examine the generation of the multiplicity of (countable) infinities to try and, by a reverse algorithm, generate a corresponding multiplicity of infinitesimalities. With any luck, they can be both integrated into a single well-behaved Ratio system which encompasses as well all Rational numbers.

In this scheme, Irrational numbers are taken as Not Numbers in principle, their Order relation to Rationals being abolished and they not being dealt with upfront; or being dealt with as a result of a more general framework of non-terminating algorithms.

Distance

Distance is the quantification or measurement of the Order relation; so, in the transitive difference abstraction, Distance means likewise the quantification or measurement of such transitive difference relation between two objects. In this way, Distance can be thought of as being as complex as the objects being related.

Distance is typically an object of the same type as the compared objects (when the compared objects are of the same type) (distance between numbers is a number, distance between Colors is a Color, etc.). But exceptions can occur and, in this case, the type may be said to be a parameterized type Distance<T>. Comparisons of objects between different types (i.e. "morphisms") are not considered at this point.

An exception to the above rule is when a distance is so small that it is "the smallest distance" before zero. (Or, in difference terms, being the "smallest difference" before equality.) This measurement transitions from being a Distance to being an Infinitesimal. All Infinitesimal Numbers are smaller than all Numbers and are larger than Zero. All Infinitesimal differences are less different that any Non-Infinitesimal differences and are less different than Equality. As such, Distance (as representing a difference) is a Sum type composed of either the type being measured or the type Infinitesimal.

For example, distances between Colors are, in pseudocode, of type Color | Infinitesimal. Distances between Lists are of type List | Infinitesimal. For types $T$ with more complex differences, their Distances may be Distance<T> | Infinitesimal.

A sum type is an algebraic type; that is, a compound type over some algebraic operation (in this case, sum).

Termination

Let's re-mention that convergence is synonym with having a limit; and non-convergence, with not having a limit.

In a computational way, an algorithm or function is said to either terminate or not terminate. Computationally, a numeric function which is divergent, for example, a function with an asymptote to the infinite, has a non-terminating enumeration; it always produces more values along its domain.

As another example, the enumeration of a numeric function which produces a finite set (that is, a function $\mathbb{B}\rightarrow\mathbb{N}$ with a finite domain) can be proved to terminate (there is no operation on a number which produces a non-finite set). Such finite functions are not convergent, nor divergent.

A function that converges is a third class of functions with respect to termination: its enumeration terminates, but only after infinite iterations.

Infinity out of finiteness?

For a function's enumeration to not terminate in a finite amount of time, the function must either operate over an infinite domain (that's trivial) and not have a limit; or transform elements from a finite domain into infinite sets.

Such a function will then be subject to the criteria of whether it converges: in the positive case its enumeration eventually terminates, and in the negative case, its enumeration is really is non-terminating.

Function enumeration

We'll consider function enumeration as a recursive process, or function $E(f)$, which takes a function as argument, aggregates the function's value for the "current" element of its domain, and calls itself with the function and "next" element in the domain as specified by the domain's order relation.

The enumeration produces a List. A list $l$'s element, denoted by its position $n$ is $l[n]$.

As such, enumeration of divergent functions is non-terminating; enumeration of finite-domain functions is finitely terminating; and enumeration of convergent functions is eventually terminating.

The key here is that the enumeration for a convergent function is terminating, which is used as a definition as there being a limit on a function with an infinite domain.

Does the function include the limit?

No. If the function included the limit in its image (as its final element), then we could say the limit of an arbitrary convergent function is its last element; or test for the value of the limit of an arbitrary convergent function by checking its last element.

This, however, (and following Mathematics), would prevent characterizing the convergent function as being composed of Infinitesimal differences part a certain point (because the last element, the limit, would have difference zero, not Infinitesimal, to itself).

This boils down to not there being a passage from an Infinitesimal to any number. As such, we can't know what the element after the last Infinitesimal is; the actual value of the limit must be found another way.

We can still test for the presence of a limit by checking for termination after infinitely many steps, though.

Final definition

With the conceptualizations in Object aspects and Termination characteristics made, it is possible to re-build Limit as:

${\color{green}L}({\color{red}D})$ is the limit of a function $f:A \rightarrow B$, respective to a certain transitive difference ${\color{red}D}(a,b)$, if after ${\color{fuchsia}E}(f)[{\color{brown}x'}]$, the difference ${\color{red}D}({\color{fuchsia}E}[{\color{brown}x}],{\color{green}L})$ is of type $\verb|Infinitesimal|$.

We can "retroactively" test for a limit in terms of the infinite-domain function's enumeration. This does not depend on the concepts of the type of the elements, or the difference between elements.

A function $f:{\color{red}A} \rightarrow B$, with infinite domain ${\color{red}A}$, has a limit if the enumeration $E(f)$ terminates.
